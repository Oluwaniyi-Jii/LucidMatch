# LucidMatch Decision Log Template

> This template documents the structure and format of decision logs generated by the LucidMatch multi-agent system. Every candidate evaluation produces a complete audit trail.

---

## Decision Log Structure

Each evaluation generates a JSON decision log with the following structure:

```json
{
  "decision_id": "uuid-v4",
  "timestamp": "ISO-8601 datetime",
  "version": "1.0",
  
  "candidate": {
    "anonymized_id": "CAND-XXXX",
    "profile_hash": "sha256-hash"
  },
  
  "job": {
    "job_id": "JOB-XXXX",
    "title": "Job Title",
    "department": "Department Name"
  },
  
  "agents": {
    "parser": { ... },
    "reasoner": { ... },
    "auditor": { ... },
    "strategist": { ... }
  },
  
  "final_decision": {
    "recommendation": "Strong Hire | Hire | Maybe | Pass",
    "match_score": 0-100,
    "confidence": 0.0-1.0,
    "flagged": true/false,
    "human_review_required": true/false
  },
  
  "audit_metadata": {
    "model_version": "claude-3.5-sonnet",
    "processing_time_ms": 8500,
    "tokens_used": 12500
  }
}
```

  "flag_type": "socioeconomic_proxy | gender_bias | age_indicator | score_variance",
  "severity": "low | medium | high | critical",

    "indicator_found": "Sailing (expensive hobby)",
    "location_in_resume": "Interests section",
    "impact_on_score": "Unknown - may have influenced evaluation",

    "status": "pending | reviewed | resolved",
    "reviewer": null,
    "reviewed_at": null,
    "action_taken": null,

## Log Query Examples

### Find all flagged decisions for a job posting
```sql
SELECT * FROM decision_logs 
WHERE job_id = 'JOB-1234' 
AND final_decision.flagged = true
ORDER BY timestamp DESC;
```

### Aggregate bias scores by time period
```sql
SELECT 
  DATE_TRUNC('week', timestamp) as week,
  AVG(agents.auditor.output.scores.fairness_score) as avg_fairness
FROM decision_logs
WHERE timestamp > NOW() - INTERVAL '90 days'
GROUP BY week
ORDER BY week;
```

### Find decisions requiring human review
```sql
SELECT * FROM decision_logs
WHERE final_decision.human_review_required = true
AND decision_id NOT IN (
  SELECT decision_id FROM human_reviews
)
ORDER BY timestamp ASC;
```
